Type Aliases

TypeScript permet de créer des alias de type pour simplifier la déclaration de types complexes. Cela peut être utile pour rendre le code plus lisible et plus facile à maintenir.

Par exemple, vous pouvez créer un alias de type pour un objet représentant une personne : 

type Person = {
    name: string;
    age: number;
}; // Alias de type pour un objet Person

Vous pouvez ensuite utiliser cet alias de type pour déclarer des variables ou des paramètres de fonction : 

let person: Person = { name: 'John', age: 30 };

Intersection

TypeScript permet de créer des types d'intersection en combinant plusieurs types en un seul. Cela permet de créer des types plus complexes en combinant plusieurs types simples. 

type Point = {
    x: number;
    y: number;
};
type Drawable = {
    draw: () => void;
}
type DrawablePoint = Point & Drawable; // Type d'intersection entre Point et Drawable

Interfaces

TypeScript permet également de définir des interfaces pour décrire la structure d'un objet. Les interfaces sont similaires aux alias de type, mais elles peuvent être étendues et implémentées par des classes.

Par exemple, vous pouvez définir une interface pour un objet représentant une personne : 

interface Person {
    name: string;
    age: number;
} // Interface pour un objet Person

Vous pouvez ensuite utiliser cette interface pour déclarer des variables ou des paramètres de fonction : 

let person: Person = { name: 'John', age: 30 };

Étendre une interface

Vous pouvez étendre une interface pour créer une nouvelle interface qui hérite des propriétés de l'interface d'origine. Cela permet de créer des types plus complexes en combinant plusieurs interfaces.

Par exemple, vous pouvez créer une interface Employee qui étend l'interface Person : 

interface Employee extends Person {
    employeeId: number;
    department: string;
} // Interface Employee qui étend l'interface Person

Propriétés optionnelles

TypeScript permet de déclarer des propriétés optionnelles dans une interface en utilisant le caractère ?. Cela signifie que la propriété peut être présente ou absente dans l'objet.

Par exemple, vous pouvez déclarer une propriété optionnelle address dans l'interface Person : 

interface Person {
    name: string;
    age: number;
    address?: string; // Propriété optionnelle
} // Interface Person avec une propriété optionnelle

Extra properties

TypeScript permet de déclarer des propriétés supplémentaires dans une interface en utilisant le caractère [key: string]: any. Cela signifie que l'objet peut avoir des propriétés supplémentaires qui ne sont pas spécifiées dans l'interface.

Par exemple, vous pouvez déclarer une propriété supplémentaire extra dans l'interface Person : 

interface Person {
    name: string;
    age: number;
    [key: string]: any; // Propriété supplémentaire
} // Interface Person avec une propriété supplémentaire

Interface vs Type Alias

Les interfaces et les alias de type sont similaires, mais il existe quelques différences clés :

  • Les interfaces peuvent être étendues et implémentées par des classes, tandis que les alias de type ne le peuvent pas.
  • Les interfaces peuvent être fusionnées, tandis que les alias de type ne le peuvent pas.
  • Les alias de type peuvent représenter des types primitifs, des tableaux et d'autres types complexes, tandis que les interfaces ne peuvent représenter que des objets.

If you would like a heuristic, use interface until you need to use features from type.
